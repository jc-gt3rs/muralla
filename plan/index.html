Part 1

Project Overview

Project Name & Translation

Tinig sa Liwanag (Translation: "Voice in the Light")

Tagline

A Filipino reading companion built for every student, engineered for those who need it most.

Mission

To make reading accessible for Filipino students across all dialects and all learning needs — with dyslexic students as the north star of every design decision.

The Problem

The Philippines has more than 130 languages. Most of them are invisible in AI and speech technology. Tagalog has seen progress, but Cebuano, Ilocano, Hiligaynon, and dozens of others have no open datasets, no benchmarks, and no usable models. Meanwhile, 1 in 5 students has a reading difficulty, and existing learning platforms were built for English-primary, high-bandwidth, neurotypical users.

A Grade 7 student in Cebu reading a Tagalog science textbook on a slow mobile connection with an undiagnosed reading difficulty has no tools. Tinig sa Liwanag is built for them.

The Two-Birds Approach

This project simultaneously satisfies two competition case tracks:

GitHub Education Case (Tinig sa Liwanag Track)
  • Curated PLD corpus subsets for 3 underrepresented dialects
  • Fine-tuned Coqui TTS / VITS model per dialect
  • Reproducible training pipeline
  • Published to HuggingFace under MIT license
  • The web application serves as production proof that the open model works
Accenture Case (AI-Powered Study Companion Track)
  • Reads and explains content in Filipino and regional languages
  • Grade-level-aware Gemini summarizer
  • Pronunciation rater gives personalized reading feedback
  • Designed for low-bandwidth and mobile-accessible use
  • Zero friction: No login, no install, works in any modern browser

Who It Is For

Primary Users: Filipino students, Grades 5-10, in any dialect region, who struggle with reading fluency, reading comprehension, or encountering academic text in an unfamiliar language. Dyslexic students are the north star — if a feature works for them, it works for everyone.

Secondary Users: Any student who wants to listen to text, get a summary in their dialect, or look up an English word; teachers seeking a low-setup reading support tool.

Part 2

Core Features

FEATURE 1: BASAHIN MO (TTS Reader)

/basahin

What it does: Paste or upload text. Pick a dialect. Press play. A clean audio reader with adjustable speed and pitch. For students who learn better by listening, or who encounter text in a language they can read but struggle to process.

Key Behaviors
  • Text input via paste or file upload (.txt, .pdf text layer)
  • Dialect selector (Filipino, Tagalog, Cebuano, Ilocano, Hiligaynon)
  • Speaking rate control (0.5x to 2.0x) and pitch control
  • Sentence-level chunked TTS — playback starts fast, bandwidth stays low
  • Session cache — replaying the same sentence uses cached audio (no repeat API calls)
  • OGG Opus encoding when browser supports it (~half MP3 file size)
Technology & Bandwidth

Google Cloud TTS API (fil-PH native voice)
Next.js API route proxy (API key stays server-side)

Bandwidth profile: Optimized (sentence chunks, cached)

★ STAR FEATURE

FEATURE 2: SAMAHAN MO AKO (Co-Reader + Pronunciation Rater)

/samahan

What it does: Text is displayed and read word by word. The current word is shown in a highlighted pill. The student taps right or uses the right arrow key to advance, taps left to go back. While the TTS reads each word, the student reads it back into the mic. The system scores pronunciation in real time and colors each word based on the result. At session end, a summary shows which words need more practice.

Key Behaviors & Highlights
  • Word-by-word display with bold pill highlight (shape + color)
  • Tap right / swipe right / right arrow = next word; left = go back
  • Mic access via Web Speech API SpeechRecognition
  • Pronunciation comparison (spoken word vs expected word) with three highlight states:
    • Green = correct pronunciation
    • Yellow = close, needs minor review
    • Red = incorrect, needs practice
  • Color + icon feedback (never color alone — colorblind safe)
  • OpenDyslexic font toggle, 3 adjustable font sizes, 3 spacing levels
  • Fallback for unsupported browsers: manual advance without scoring (compatibility notice shown)
Technology & Bandwidth

Web Speech API SpeechRecognition (on-device, zero bandwidth)
Google Cloud TTS API
React state machine for word cursor

Bandwidth profile: Zero for pronunciation scoring (on-device)
Requirements: Chrome or Edge for pronunciation scoring

FEATURE 3: ANO ANG SALITA (Spelling & Dictionary Lookup)

/salita

What it does: Type any English word. Get the definition, word type, IPA pronunciation, and an example sentence. Reads the definition aloud on request. Cached locally so repeat lookups are instant and offline.

Key Behaviors
  • Text input with instant lookup on submit
  • Returns: word type, definition, IPA pronunciation, example sentence
  • Optional TTS reads the definition aloud
  • Each looked-up word is cached in localStorage
  • Second and subsequent lookups for the same word are instant and require no connection
  • Graceful error state for words not found
Technology & Bandwidth

Free Dictionary API (dictionaryapi.dev) — no API key needed
Google Cloud TTS API for read-aloud
localStorage for offline word cache

Bandwidth profile: Minimal (JSON response, locally cached)

FEATURE 4: IPALIWANAG MO (Dialect Summarizer)

/ipaliwanag

What it does: Paste a textbook excerpt, article, or assignment. Pick a dialect and a grade level. Gemini summarizes it in plain, student-friendly language in that dialect. Output can be read aloud via TTS. The only tool of its kind for Philippine regional languages.

Key Behaviors & Prompt Engineering
  • Text input (capped at 3,000 characters for demo tier)
  • Dialect selector and Grade level selector (Grades 4-6, Grades 7-9, Grades 10-12)
  • Gemini 1.5 Flash generates dialect-appropriate summary
  • Read aloud button on output (routes to TTS)
  • Prompt engineering per dialect:
    • Filipino: "Summarize in simple Filipino. Use words a Grade 5-8 student would understand."
    • Cebuano: "Summarize in simple Cebuano (Bisaya). Use vocabulary a student from Visayas/Mindanao would understand."
    • Ilocano: "Summarize in simple Ilocano. Use vocabulary a student from the Ilocos region would understand."
    • Hiligaynon: "Summarize in simple Hiligaynon. Use vocabulary a student from Western Visayas would understand."
Technology & Bandwidth

Gemini 1.5 Flash via @google/generative-ai
Google Cloud TTS for read-aloud
Next.js API route proxy

Bandwidth profile: Minimal (text only, response < 2KB)

TTS Synthesis Dialect Note

Google Cloud TTS has native fil-PH support (Filipino and Tagalog). For Cebuano, Ilocano, and Hiligaynon, the text is pre-processed through Gemini to normalize pronunciation-consistent spelling before synthesis via the Filipino voice. This is a known approximation. The open-source Coqui model artifact is built specifically to close this gap in the research community.

Design Language & System

The interface uses clear, bright, warm tones tailored for accessibility and dyslexic readability, avoiding harsh high contrasts while ensuring WCAG compliance.

Color Palette

Sun Yellow #F7C948

Primary interactive element highlights and active focuses.

Sun Light #FFF5CC

Warm yellow card and text container backgrounds.

Sky Blue #2D6BE4

Hyperlinks, primary labels, and callout border accents.

Ink Dark #1A1A2E

Primary body text, code blocks, and primary layouts.

Paper #FAFAF5

Overall page background (warm off-white to prevent eye strain).

Success Green #16A34A

Correct pronunciation feedback styling.

Warning Amber #D97706

Close attempt pronunciation feedback.

Danger Red #DC2626

Incorrect pronunciation feedback styling.

Typography System

  • Default Body Font: Atkinson Hyperlegible. Designed specifically for low-vision readers to maximize character recognition. Default for all screens.
  • Co-reader Font Option: OpenDyslexic. Features weighted bottoms to prevent rotation and flipping confusion in dyslexic readers.
  • Display & Headings: Syne (800 weight). A strong, modern display font.
  • Monospace elements: System Courier New (or JetBrains Mono). Used for paths, structures, and keys.

Reading Surface Rules

  • Minimum text/body size: 18px
  • Line height: 1.75
  • Letter spacing: Normal default, +0.05em in regular co-reader mode, and +0.12em in high-spacing mode.
  • Word highlight style: Bold filled pill (shape + color). Never rely on underlines or colors alone.

Accessibility Constraints

  • Pronunciation feedback must always combine color and icon indicator assets, never color alone.
  • Respected motion reduction queries (prefers-reduced-motion).
  • Full keyboard navigation support on the co-reader layout.
  • Passes WCAG AA minimum contrast ratio across all elements.
  • High-visibility focus rings on all active elements.
Part 7

Low-Bandwidth Strategy

The Philippines has regional connectivity gaps that most applications ignore. Every core feature in Tinig sa Liwanag is optimized as a hard constraint, not an afterthought.

Feature-by-Feature Bandwidth Architecture

Pronunciation Rater (Cost: Zero)

Web Speech API SpeechRecognition is executed entirely inside the browser's engine. No voice audio is sent to the server for evaluation. Scoring works offline.

Dialect Summarizer (Cost: Minimal)

Text-only responses. Gemini payloads are stripped and returned as pure string formats, ensuring transaction payloads are capped below 2KB.

Dictionary Lookup (Cost: Single-Fetch)

Every looked-up word details are saved to localStorage. Subsequent definitions of the same words are loaded locally without trigger network queries.

Text-to-Speech Audio (Cost: Optimized)

Synthesized in sentence chunks. One sentence is preloaded in the background while the current is playing. Opus OGG compression is favored to reduce download sizes by ~50% compared to standard MP3 codecs. Replaying a sentence loads from the browser cache.

Assets & Bundle Strategy
  • Fonts: Self-hosted, cached on first visit. Styled with font-display: swap to render text instantly.
  • JS Bundling: App Router enables automated route-level code splitting. Readers do not load summarizer libraries.
  • PWA: manifest.json and service workers load the core app shell instantly even on spotty networks.

Part 9

Known Constraints & honest Limitations

A transparent disclosure of the system's operational boundaries in MVP form:

1. Google TTS Dialect Approximation

Google TTS does not natively support Cebuano, Ilocano, or Hiligaynon. They are approximated by spelling text phonetic-consistently via Gemini before sending to the Filipino voice. Output quality is functional, but imperfect; the custom Coqui model is the long-term solution.

2. Pronunciation Rater Browser Requirements

The Web Speech API's SpeechRecognition is only fully implemented in Google Chrome and Microsoft Edge. Users on Safari or Firefox will receive a notice banner and fallback to reading without scoring.

3. Summarizer Input Limits

Payload inputs are limited to 3,000 characters per summarize query. This represents a security filter for rate limits during the hackathon tier, not a system design limit.

4. Minimal Data Privacy Footprint

No account setups, no server tracking databases, and zero tracking scripts are used. Microphone audio is kept local in the browser runtime; vocabulary data remains inside the user's localStorage sandbox.

5. Regional Accent Gaps

The synthetic fil-PH voice will sound distinct from native Cebuano or Ilocano accents. This is highlighted to the student. It works for standard learning, and the open-source pipeline is intended to resolve this.

Part 11

Execution Strategy

Build Order Rationale

The build sequence is designed to establish solid dependencies first, ensuring a demonstrable application state is met at the end of each day:

  • Day 1: TTS API & Reader — Solves the primary external API dependency first. TTS Reader is the simplest integration consumer, verifying basic audio pipelines.
  • Day 2: Co-Reader Core — Core features are complex; prioritizing highlight states, font overrides, and keyboard navigation before adding mic capture simplifies debugging.
  • Day 3: Pronunciation Rater & Auxiliary Services — Adds Web Speech APIs on top of the tested Day 2 foundation. Summarizer and dictionary routes run in parallel.
  • Day 4: Polish & Documentation — Feature freeze. Focus entirely on layout tests, accessibility checks, and repository publication.

Parallel Work Distribution

Person A: Feature Lead
  • Day 1: TTS APIs, proxy hooks, and base audio players.
  • Day 2: Tokenization cursors, word highlighters, and per-word TTS play.
  • Day 3: SpeechRecognition integrations and scoring comparisons.
  • Day 4: Keyboard fixes, accessibility checks, and demo production.
Person B: UI/UX Lead
  • Day 1: Layout, headers, landing routes, and dialect picks.
  • Day 2: OpenDyslexic settings, spacing, size bars, and samahan layouts.
  • Day 3: Dictionary, summarizer dashboard, and Grade level pickers.
  • Day 4: Swatch alignments, mobile adjustments, and submission files.

Project Risk Register

Risk Vector Likelihood / Impact Mitigation Strategy
Google TTS Key Issue Medium / High Test connection strings on Day 0. Fallback: browser-native SpeechSynthesis if keys are blocked.
Web Speech API Compatibility Low-Medium / Medium Target demo flows to Chrome/Edge. Render compatibility notices on unsupported browsers.
Gemini Output Variant Quality Medium / Medium Iterate and fix prompts early. Use Filipino and Tagalog as primary demo scripts.
Vercel CI/CD Build Failures Low / High Deploy a skeleton app on Day 0 to establish clean routing integrations early.
Timeline Overrun on Co-Reader Medium / Medium Build dictionary page as simple fetch-render. Strip summarizer grade filters if needed.

Live Demonstration Script (5-7 mins)

  1. Landing Page (30s): Introduce the tagline, name the regional dialect gap, and state the accessibility focus.
  2. Ipaliwanag Mo (90s): Paste text. Summarize to Cebuano for Grade 7-9. Play the summary result via TTS.
  3. Samahan Mo Ako (2m): Highlight co-reader controls. Toggle OpenDyslexic mode. Speak into the mic to show correct (green) and incorrect (red) pronunciation, followed by the session recap.
  4. Basahin Mo (45s): Paste longer texts. Demonstrate speech rates, sentence chunking, and dialect selectors.
  5. Ano ang Salita (30s): Look up a term to showcase definitions and offline localStorage loads on second lookup.
  6. Closing (30s): Reiterate zero login, PWA support, and link to public GitHub/HuggingFace model checkpoints.
