The Problem

A gap in language, technology, and learning support

130+

Philippine languages spoken across the archipelago

1 in 5

Students affected by dyslexia and reading difficulties

~4

Philippine languages with meaningful open speech technology support

Despite the Philippines being home to more than 130 languages, most regional languages remain invisible in AI and speech technology. Tagalog has seen progress, but Cebuano, Ilocano, Hiligaynon, and dozens of others lack open datasets, benchmarks, and models. Meanwhile, millions of Filipino students navigate classrooms with no accessible reading support in their mother tongue.

Existing learning platforms are built for English-primary, high-bandwidth, and neurotypical users. They were not designed for a Grade 7 student in Cebu reading a Tagalog science textbook on a slow mobile connection with an undiagnosed reading difficulty.

Tinig sa Liwanag addresses both gaps simultaneously: a browser-based reading companion for Filipino students, and an open-source speech infrastructure contribution for Philippine languages that researchers and developers can build on.

Who It's For

Designed for everyone. Engineered for those who need it most.

Tinig sa Liwanag is not a dyslexia-only tool. Any Filipino student benefits from dialect-aware TTS, dialect summarization, and a co-reader. But every design decision is tested against one question first: does this work for a student with dyslexia? If it does, it works for everyone.

Primary user: Filipino student, Grades 5–10, any dialect region, who struggles with reading fluency, reading comprehension, or encountering academic text in an unfamiliar language.

Secondary users: Any student who wants to listen to text, get a summary in their dialect, or look up an English word. Teachers who want a low-setup reading support tool for their class.

The Features

Five tools. One URL. No login.

/basahin
Basahin Mo
TTS Reader

Paste or upload text. Pick a dialect. Press play. A clean audio reader with adjustable speed and pitch. Sentence-level chunked playback keeps bandwidth low and lets the student follow along.

Google Cloud TTS 5 dialects Speed control Low bandwidth
/samahan
Samahan Mo Ako
Co-Reader + Pronunciation Rater

Text is read word by word, with the current word highlighted in a bold pill. Tap right to advance, tap left to go back. OpenDyslexic font toggle, adjustable spacing and size. While TTS reads each word, the student reads it back — the mic scores pronunciation in real time and colors each word green, yellow, or red.

Web Speech API OpenDyslexic font Correct Close Needs practice
/salita
Ano ang Salita
Spelling & Dictionary Lookup

Type any English word. Get the definition, word type, IPA pronunciation, and an example sentence. Optional TTS reads the definition aloud. Results are cached locally so repeated lookups are instant and offline.

Free Dictionary API IPA pronunciation No API key needed Offline cache
/ipaliwanag
Ipaliwanag Mo
Dialect Summarizer

Paste a textbook excerpt, article, or assignment. Pick a dialect and a grade level. Gemini summarizes it in plain, student-friendly language in that dialect. Output can be read aloud via TTS. The only tool of its kind for Philippine regional languages.

Gemini 1.5 Flash 5 dialects Grade-level aware TTS output
Feature Deep Dive

The Pronunciation Rater

This is the feature that turns a passive reading tool into an active reading tutor. Here is how it works in sequence:

The co-reader highlights a word and the TTS voice reads it aloud. The student then reads the same word back into their microphone. The browser's Web Speech API transcribes the student's spoken word and compares it to the expected word. The highlight updates immediately based on the result.

At the end of a session, a score summary shows the student which words need more practice. Color feedback always pairs with an icon so it is never color-alone — accessible for colorblind users.

Browser note: Web Speech API requires Chrome or Edge. A compatibility notice is shown to students on unsupported browsers, with a fallback to manual tap-to-advance without scoring.

Low-Bandwidth Design

Designed for Philippine internet realities

No feature is sacrificed for low bandwidth. The architecture is designed around it.

Zero bandwidth
Pronunciation Rater

Web Speech API runs entirely on-device. No audio is ever sent to a server.

Minimal bandwidth
Summarizer

Text-in, text-out. Even on 2G, Gemini responses are under 2KB.

Cached locally
Dictionary Lookup

Each word cached in localStorage. Second lookup is instant and offline.

Optimized
TTS Audio

Synthesized sentence-by-sentence. Playback starts immediately. One sentence ahead is preloaded. OGG Opus encoding (~half of MP3 size) used when the browser supports it.

One-time load
Fonts & App Shell

Fonts are self-hosted and cached. Service worker caches the app shell after first visit.

Route-split
JavaScript Bundle

Next.js App Router splits code per route. A student using only the co-reader never downloads the summarizer code.

Open Source Artifact

The GitHub Education deliverable

Separate from the app, the team publishes a reusable open-source artifact that future researchers and developers can extend. This is what directly satisfies the GitHub Education case requirement for "foundational infrastructure."

📚 HuggingFace Dataset + Model
Published under MIT license, separate from the app repository
  • ▶ Curated and formatted subset of the Philippine Languages Database (PLD) for Cebuano, Ilocano, and Hiligaynon — cleaned, normalized, and ready for TTS training
  • ▶ Fine-tuned VITS / Coqui TTS model checkpoint per dialect, with model card and training notes
  • ▶ Reproducible training pipeline in Python with documented steps — any researcher can retrain or extend
  • ▶ HuggingFace dataset card with language metadata, speaker demographics, and known limitations
  • ▶ The app is the demonstration that the open model works in a real product — closing the loop between research artifact and usable technology
Design Language

Warm, accessible, student-first

Every visual decision is tested against the dyslexic student first. If it works for them, it works for everyone.

  • 💡Atkinson Hyperlegible as the default body font — designed specifically for low-vision and reading-difficulty users, open source via Google Fonts.
  • 💡OpenDyslexic font available as a toggle in the co-reader — extra-weighted bottoms help prevent letter rotation confusion.
  • 💡Minimum 18px on all reading surfaces. Generous line height (1.75). Extra letter spacing in co-reader mode.
  • 💡Word highlight is a bold pill, never just an underline or color change alone. Shape + color.
  • 💡Pronunciation feedback uses color + icon — never color alone. Colorblind-safe by design.
  • 💡Filipino sun yellow #F7C948 as the primary interactive color. Warm, energetic, culturally grounded.
  • 💡No modals, no popups, no distracting animations on reading surfaces. Motion is reserved for feedback moments only.
Known Constraints

Honest limitations

  • ⚠️Google TTS has native support for Filipino only. Cebuano, Ilocano, and Hiligaynon are approximated by pre-processing text through Gemini for pronunciation-consistent spelling before synthesis. This is a known limitation the open-source model artifact is built to address.
  • ⚠️The Web Speech API (pronunciation rater) requires Chrome or Edge. Firefox and Safari users see a compatibility notice and fall back to manual tap-to-advance co-reading without scoring.
  • ⚠️The Gemini summarizer requires a live connection and caps input at 3,000 characters to stay within Flash rate limits during the demo. This is a demo constraint, not a fundamental limitation.
  • ⚠️No user data is stored. Mic access is session-only and never sent to a server. Dictionary cache is browser-local. No account, no tracking.
